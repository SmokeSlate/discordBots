Features

Core systems

High-level tour of the systems inside main.py.

Ticket system

Private threads with category select and staff controls.

  • Custom categories defined via slash commands.
  • Ephemeral confirmations; duplicate ticket prevention per user.
  • Stats, open lists, and persistent close/staff note buttons.

Snippets + modals

Slash-managed snippets that still fire from a prefix.

  • Static/dynamic with placeholders ({1}) and {ping}.
  • Default form editor (/editsnippet) for multiline copy.
  • Per-guild snippets that stay consistent between restarts.

Auto replies

Regex or contains matching with smart filters.

  • Role/channel allow + deny lists and case sensitivity for contains.
  • Cooldown scopes: guild, channel, user, member, role, category, thread.
  • Send text responses or hand off to a snippet; migrates older configs automatically.

Giveaways

Scheduled giveaways with button entries.

  • /giveaway posts an embed with a join button and optional role requirement.
  • Winners are selected on schedule with host mentions and entry counts.
  • Persistent join view and embed updates survive restarts.

Pins + reaction roles

Keep key info at the bottom and map reactions to roles.

  • /setpin reposts the tracked message to stay at the bottom.
  • /reactionrole + removal for emoji-to-role mapping on any message.
  • Permission-aware feedback so staff know when Discord blocks actions.

Moderation + cleanup

Slash-first moderation with precise filters.

  • Timeouts, kicks, bans, unbans, slowmode.
  • /clear filters users, roles, attachments, ranges, and bot-only cleanup.
  • Role management including /addroleall with optional bot exclusion and safe errors.
